# comorebi studio - Claude Code 運用ルール

## プロジェクト概要
こもれびスタジオ（ブランディング支援会社）の育成プログラム管理。
担当: よっしーさん（川原さん）が若手メンバーの研修・セッションを週1回実施。

## メンバー
- **はるかちゃん**: アルバイト / 週1セッション / 3ヶ月プログラム（全8回）
- **ウェラー**: ブランドマネージャー / 2026年4月10日〜開始

## ディレクトリ構造
```
01_training/members/{名前}/
  daily-reports/   ← 日報テキスト（YYYY-MM-DD.txt）
  sessions/        ← セッション記録
  presentations/   ← プレゼン資料
02_marketing/      ← 集客・マーケティング
03_academy/        ← アカデミー構想
docs/meeting-notes/ ← 会議メモ
index.html         ← 育成ダッシュボード（単一HTMLファイル）
```

## 自動業務フロー

### トリガーワード: 「セッション終わった」「研修終わった」「セッション完了」
以下を **すべて自動で** 実行する。途中で確認を求めず、最後にまとめて報告する。

#### Step 1: Gmailから議事録を取得
- `mcp__claude_ai_Gmail__gmail_search_messages` で直近のGeminiメモを検索
- 検索クエリ: Google Meet終了後にGeminiが自動送信する議事録メール（「Gemini」「メモ」等で検索）
- 該当メールの本文（議事録・文字起こし）を `gmail_read_message` で取得
- Google ドキュメントのURLが含まれている場合は `WebFetch` でドキュメント内容も取得を試みる

#### Step 2: セッション記録を保存
- 取得した内容をテキストファイルに整形
- `01_training/members/{名前}/sessions/` に `{回数}_{テーマ}.txt` として保存
- 生の文字起こしデータも必要に応じて保存

#### Step 3: 分析レポートを作成
以下の観点で分析し、レポートを出力する:

1. **成長ポイント** — 前回と比較して何が変わったか、どんな気づきがあったか
2. **課題・伸びしろ** — まだ取り組めていないこと、次に伸ばすべきポイント
3. **本質的な変化** — 表面的な知識習得ではなく、思考や姿勢の変化を読み取る
4. **次回セッションへの問いかけ案** — よっしーさんのスタイル（教えるのではなく問いかける対話型）に合わせた質問を3つ以上提案

#### Step 4: ダッシュボード（index.html）を更新
- セッション履歴に新しいエントリを追加
- 「本質的な変化」セクションに新しいWeekを追加
- セッション進捗（回数/8）を更新
- 強み・課題リストに新たに発見されたものがあれば追加
- 次回セッション情報を更新
- 「最終更新」日付を更新
- 次回セッションまでのアクションを更新
- 総合所見を最新の状態に更新

#### Step 5: コミット
- 変更をgitコミット（ユーザーに確認してからコミット）

---

### トリガーワード: 「日報来た」「日報」
1. Gmailまたはダウンロードフォルダから日報テキストを取得
2. `daily-reports/YYYY-MM-DD.txt` として保存
3. 日報の内容を分析:
   - 1週間の変化・気づき
   - 成長の兆し
   - フォローすべきポイント
4. 次回セッションで触れるべき話題を提案

---

### トリガーワード: 「ダッシュボード見せて」「進捗確認」
- 現在のダッシュボード（index.html）の内容をサマリーとして報告
- 各メンバーの進捗状況、次回セッション情報、直近の成長ポイントを簡潔に伝える

## 分析の方針
- よっしーさんの育成哲学: **教えるのではなく、問いかけて本人に気づかせる**
- ブランディング・マーケティングは「人間・社会・環境」の上に成り立つ
- 表面的なスキル習得よりも、**思考のOS（自己認知・視点の変化）** の変化を重視
- 分析では「川原」タグ（よっしーさんの観察）と「AI分析」タグを区別する
- 過去のセッション記録・日報との **変化の連続性** を必ず読み取る

## メモリ自動保存ルール
セッション分析・日報分析のたびに、以下を memory/ に自動保存する:
- メンバーの成長フェーズの変化（どの段階にいるか）
- 新たに発見された強み・課題
- よっしーさんが下した重要な判断や方針変更
- 会話の中で「覚えておいて」と言われたこと

既存メモリと重複する場合は上書き更新する。新規作成はしない。

## 行動方針
- やるべきことがあれば確認を待たず積極的に進める
- 判断に迷う場合のみ確認する
- 分析→保存→ダッシュボード更新は一気通貫で止めない

## 出力スタイル
- 簡潔に。冗長な前置きは不要
- 分析結果は箇条書きベースで読みやすく
- 問いかけ案は、はるかちゃんやウェラーが「自分で考えたくなる」ような問いにする
- ダッシュボード更新時は差分だけ報告する（全文読み上げない）
